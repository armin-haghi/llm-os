# Change Policy

## Local experiment
A pattern used in one project or one milestone only.

## Candidate pattern
A pattern reused enough to compare against alternatives and consider
promotion.

## Operating standard
A pattern promoted into `core/`, `skills/`, or templates because it reduces
confusion across projects.

## Promotion rule
Promote only when reuse is proven and the added standard reduces confusion or
decision load.

## Deprecation rule
Deprecate before removing when changing a core concept that other surfaces
still reference.

## Update rule
When a core concept changes, update impacted layers together so entrypoints,
templates, and skills do not drift.
